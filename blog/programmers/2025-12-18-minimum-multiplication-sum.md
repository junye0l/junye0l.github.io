## 문제 설명

길이가 같은 두 배열 A, B에서 각각 한 개씩 숫자를 뽑아 곱한 값을 누적하여 더할 때, 최종 누적값이 최소가 되도록 하는 문제입니다.

**제약 조건**
- 각 배열에서 한 번 뽑은 숫자는 다시 뽑을 수 없습니다.
- 배열 A, B의 크기: 1,000 이하의 자연수
- 배열 A, B의 원소의 크기: 1,000 이하의 자연수

**예시**
```
A = [1, 4, 2]
B = [5, 4, 4]

1 × 5 = 5
4 × 4 = 16
2 × 4 = 8
합계: 29
```

<br/>

## 문제 접근

이 문제의 핵심은 **두 수의 곱셈 합을 최소화**하는 것입니다.

곱셈의 특성을 생각해보면:
- 큰 수 × 큰 수 = 매우 큰 값
- 작은 수 × 작은 수 = 작은 값
- **큰 수 × 작은 수 = 중간 값**

따라서 최솟값을 만들기 위해서는:
- A 배열의 **가장 작은 수**와 B 배열의 **가장 큰 수**를 곱하고
- A 배열의 **두 번째로 작은 수**와 B 배열의 **두 번째로 큰 수**를 곱하는 방식으로
- 반대 방향으로 매칭하면 됩니다.

<br/>

## 해결 방법

1. 배열 A는 **오름차순**으로 정렬
2. 배열 B는 **내림차순**으로 정렬
3. 같은 인덱스끼리 곱해서 합산

<br/>

## 풀이 코드

```javascript
function solution(A, B) {
    // A는 오름차순, B는 내림차순 정렬
    A.sort((a, b) => a - b);
    B.sort((a, b) => b - a);

    let answer = 0;

    // 같은 인덱스끼리 곱해서 누적
    for (let i = 0; i < A.length; i++) {
        answer += A[i] * B[i];
    }

    return answer;
}
```

<br/>

## 다른 풀이 - reduce 사용

```javascript
function solution(A, B) {
    A.sort((a, b) => a - b);
    B.sort((a, b) => b - a);

    return A.reduce((acc, cur, idx) => acc + cur * B[idx], 0);
}
```

<br/>

## 시간 복잡도

- 정렬: O(n log n)
- 순회: O(n)
- **전체: O(n log n)**

배열의 크기가 최대 1,000이므로 충분히 효율적입니다.

<br/>

## 핵심 포인트

1. **그리디 알고리즘**: 각 단계에서 최선의 선택(작은 수 × 큰 수)을 하면 전체 최적해를 얻을 수 있습니다.

2. **정렬의 활용**: 문제를 단순화하기 위해 정렬을 활용하는 것이 핵심입니다.

3. **수학적 사고**: 곱셈의 특성을 이해하고 최솟값을 만드는 조합을 논리적으로 도출해야 합니다.

<br/>

## 예시 검증

```
A = [1, 4, 2] → 정렬 후: [1, 2, 4]
B = [5, 4, 4] → 정렬 후: [5, 4, 4]

1 × 5 = 5
2 × 4 = 8
4 × 4 = 16
합계: 29
```

```
A = [1, 2] → 정렬 후: [1, 2]
B = [3, 4] → 정렬 후: [4, 3]

1 × 4 = 4
2 × 3 = 6
합계: 10
```

## 결론

이 문제는 정렬과 그리디 알고리즘의 기본 개념을 이해하고 있는지 확인하는 문제입니다. 복잡한 알고리즘보다는 문제의 본질을 파악하고 수학적 사고를 통해 최적의 전략을 도출하는 것이 중요합니다.
